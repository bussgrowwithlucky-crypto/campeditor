# campeditor

Web UI for turning landscape clips into 9:16 viral shorts:

- Upload landscape video
- Trim a slice by start/end seconds, or let AI select the viral clip
- Auto-caption via Groq Whisper (with per-word timestamps)
- Two-line title overlay — manual input OR auto-generated by an LLM
- LLM picks which words to highlight in `#CD1111` red
- Optional color grade (None / Cinematic / Warm / Cool / Punchy)
- Render 9:16 vertical MP4, ready to download

No local models run on your PC — only FFmpeg and cloud APIs.

## Setup

Requires Python 3.11+, FFmpeg on PATH.

```powershell
cd C:\campeditor
.\scripts\run.ps1
```

The first run creates `.venv`, installs dependencies, then starts uvicorn on `http://127.0.0.1:8000`.

Open `http://127.0.0.1:8000` in your browser.

## Environment

Keys live in `.env` (already created). The current values were transmitted in chat and **must be revoked**:

- Groq: https://console.groq.com/keys
- LLM router (byNara): https://router.bynara.id/settings — requires linking Telegram + joining
  required channels, otherwise every request 403s with `telegram_required`.

Replace the values in `.env` and restart.

All variables (defaults in `app/config.py`):

| Variable | Default | Purpose |
| --- | --- | --- |
| `GROQ_API_KEY` | — | Whisper transcription |
| `GROQ_TRANSCRIPTION_MODEL` | `whisper-large-v3-turbo` | Groq model |
| `LLM_API_KEY` | — | Title and clip-selection LLM |
| `LLM_BASE_URL` | `https://router.bynara.id/v1` | Any OpenAI-compatible chat completions endpoint |
| `LLM_MODEL` | `claude-sonnet-4.5` | Model name on that endpoint |
| `LLM_FALLBACK_MODEL` | — | Optional second model tried if the primary fails |
| `FFMPEG_PATH` / `FFPROBE_PATH` | `ffmpeg` / `ffprobe` | Override if not on PATH |
| `YTDLP_COOKIES_FILE` | — | Optional Netscape cookies.txt for YouTube reference downloads |
| `YTDLP_COOKIES_FROM_BROWSER` | — | Disabled; browser cookie scraping is not used |
| `BROLL_LIBRARY_DIR` | `data/broll_library` | Legacy local library path; not used for B-roll sourcing |
| `YOUTUBE_DATA_API_KEY` / `_2` | — | YouTube Data API v3 keys for B-roll search (key rotation on 403/429), falling back to yt-dlp ytsearch when unset/exhausted |
| `BROLL_QUERY_MAX_WORDS` | `8` | Trims B-roll search queries to at most this many words |
| `BROLL_LOCAL_MATCH_THRESHOLD` | `0.35` | Local-library match score floor (0..1); below this a span falls through to the YouTube rung |
| `CAMPEDITOR_DATA_DIR` | `data` | Job storage |
| `CAMPEDITOR_MAX_UPLOAD_MB` | `500` | Upload cap |
| `CAMPEDITOR_AUTO_CLIP_SCAN_SECONDS` | `600` | Max uploaded-video seconds to analyze for AI clip selection |
| `CAMPEDITOR_AUTO_CLIP_TARGET_SECONDS` | `15` | Preferred AI-selected clip length |
| `CAMPEDITOR_AUTO_CLIP_MIN_SECONDS` | `8` | Minimum AI-selected clip length |
| `CAMPEDITOR_AUTO_CLIP_MAX_SECONDS` | `30` | Maximum AI-selected clip length |

## Known limits

- Caption line wrap is approximate (~22 chars per line, 2 lines per cue). Long unbreakable words may overflow.
- Title LLM may mis-format JSON — fallback always returns *something*.
- No word-level highlights if Groq returns no `words` — falls back to a simple heuristic.
